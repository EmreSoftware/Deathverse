# Contributing to Deathverse

Thanks for your interest in improving **Deathverse**.

By contributing, you agree to follow the guidelines below and that your contributions will be licensed under the **GNU GPL v3.0** (or any later version, at your option), the same license as this project.

---

## What You Can Contribute

- Bug fixes
- Performance improvements
- Compatibility improvements (DarkRP, TTT, weapons, etc.)
- Safe config additions that don’t break existing behavior
- Code cleanup, refactoring, and optimization
- Documentation improvements

---

## What You Should NOT Submit

- Malicious code, backdoors, “phone-home” behavior, or credential/secret harvesting
- Anything intended to bypass server permissions, exploit players/servers, or enable cheating
- Breaking changes or major redesigns **without** prior discussion
- Bundling third-party code/assets you don’t have rights to redistribute under GPL-compatible terms

## Security Issues

If you find a security vulnerability, **do not** open a public issue.
Use the repo’s **Security → Report a vulnerability** feature instead.

---

## How to Contribute (Workflow)

1. Fork the repository on GitHub.
2. Create a branch (`fix/...`, `perf/...`, `docs/...`).
3. Make your changes with clear, minimal commits.
4. Open a Pull Request with:
   - what you changed
   - why it’s needed
   - how to test it
5. Keep changes focused. Large changes should be discussed in an Issue first.
